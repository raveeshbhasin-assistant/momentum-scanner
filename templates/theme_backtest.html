Method notes.
Snapshot cadence — 09:45, 10:15, 10:45, 11:15, 11:45, 13:30, 14:00, 14:30, 15:00, 15:30. Every 30 min outside the 12:00-13:30 lunch dead-zone. This models how a trader checks the scanner, not tick-level omniscience.
Entry — at the close of the first 5-min bar ≥ the earliest qualifying snapshot.
Stop / target — bucket-specific from the signal itself: theme leader 2.0% stop / +5.0% target, gap+news 2.5% / +6.25%, low-float 3.5% / +8.75%. Target is always 2.5× stop, so an R-multiple of +2.5 = target hit exactly.
Exit — walk 5m bars forward. If bar.low ≤ stop → LOSS at stop. If bar.high ≥ target → WIN at target. Else EOD mark-to-market at 15:55. On a bar that touches both, LOSS wins (conservative).
What's different from v3.4.2 backtest — v3.4.2 tests the momentum scanner's fire-now-at-scan-time trigger on the S&P 500 universe; this tests the ThemeHunter's theme-RS/low-float scoring on a superset universe with a snapshot cadence.
